◆ am_devices_mspi_rm69330_init()

uint32_t am_devices_mspi_rm69330_init ( uint32_t  ui32Module,
am_devices_mspi_rm69330_config_t *  pDevCfg,
void **  ppHandle,
void **  ppMspiHandle 
)

Initialize the MSPI_RM69330 driver.

Parameters
ui32Module- MSPI module.
pDevCfg- device configuration.
ppHandle- MSPI module handle
ppMspiHandle- MSPI handle's handle.

This function should be called before any other am_devices_MSPI_RM69330 functions. It is used to set tell the other functions how to communicate with the external screen hardware.

The pfnWriteFunc and pfnReadFunc variables may be used to provide alternate implementations of SPI write and read functions respectively. If they are left set to 0, the default functions am_hal_iom_spi_write() and am_hal_iom_spi_read() will be used.

Returns
None.

uint32_t am_devices_mspi_rm69330_nonblocking_write ( void *  pHandle,
const uint8_t *  pui8TxBuffer,
uint32_t  ui32NumBytes,
bool  bWaitForCompletion,
bool  bContinue 
)

Programs the given range of display addresses.

Parameters
pHandle- mspi handle
pui8TxBuffer- Buffer to write the data from
ui32NumBytes- Number of bytes to write to the display memory
bWaitForCompletion- Waits for CQ/DMA to complete before return.
bContinue- memories write or write continue.

This function uses the data in the provided pui8TxBuffer and copies it to the external flash at the address given by ui32WriteAddress. It will copy exactly ui32NumBytes of data from the original pui8TxBuffer pointer. The user is responsible for ensuring that they do not overflow the target flash memory or underflow the pui8TxBuffer array

Returns
32-bit status

ui32StatusSetClr- CQ Set/Clear condition after execution.
