uint32_t *  pui32Src,
uint32_t  ui32Offset,
uint32_t  ui32NumWords 
)

This programs up to N words of the Main array on one MRAM.

Parameters
ui32InfoKey- The programming key, AM_HAL_MRAM_INFO_KEY.
pui32Src- Pointer to word aligned array of data to program into INFO0
ui32Offset- Pointer to the word aligned INFO0 offset where programming of the INFO0 is to begin.
ui32NumWords- The number of words to be programmed.

This function will program multiple words in INFO0

Returns
0 for success, non-zero for failure. 1 ui32InfoKey is invalid. 2 ui32Offset is invalid. 3 addressing range would be exceeded. That is, (ui32Offset + (ui32NumWords * 4)) is greater than the last valid address. 4 pui32Src is invalid. 5 pui32Src is invalid. 6 Flash controller hardware timeout.

Read INFO data.

This function implements a workaround required for Apollo4 B0 parts in order to accurately read INFO space.

Parameters
ui32InfoSpace- 0 = Read INFO0. 1 = Only valid to read the customer visible area of INFO1. If INFO1, the ui32WordOffset argument must be 0x480 or greater (byte offset 0x1200).
ui32WordOffset- Desired word offset into INFO space.
ui32NumWords- The number of words to be retrieved.
pui32Dst- Pointer to the location where the INFO data is to be copied to.
Returns
0 for success, non-zero for failure.

This Fills up to N words of the Main MRAM.

Parameters
ui32ProgramKey- The programming key, AM_HAL_MRAM_PROGRAM_KEY.
ui32Value- 32-bit data value to fill into the MRAM
pui32Dst- Pointer to the 4 words (16 byte) aligned MRAM location where programming of the MRAM is to begin.
ui32NumWords- The number of words to be programmed. This MUST be a multiple of 4 (16 byte multiple)

This function will fill multiple 16 byte tuples in main MRAM with specified pattern.

Note
THIS FUNCTION ONLY OPERATES ON 16 BYTE BLOCKS OF MAIN MRAM. The pDst MUST be 4 words (16 byte) aligned, and ui32NumWords MUST be multiple of 4.
Returns
0 for success, non-zero for failure. -1 invalid alignment for pui32Dst or ui32NumWords not 16 byte multiple 1 ui32InfoKey is invalid. 2 ui32Offset is invalid. 3 addressing range would be exceeded. That is, (ui32Offset + (ui32NumWords * 4)) is greater than the last valid address. 4 pui32Src is invalid. 5 pui32Src is invalid. 6 Flash controller hardware timeout.

This programs up to N words of the Main MRAM.

Parameters
ui32ProgramKey- The programming key, AM_HAL_MRAM_PROGRAM_KEY.
pui32Src- Pointer to word aligned array of data to program into the MRAM.

That is, (pui32Dst + (ui32NumWords * 4)) is greater than the last valid address. 4 pui32Src is invalid. 5 pui32Src is invalid. 6 Flash controller hardware timeout.

◆ am_hal_mram_recovery()

int am_hal_mram_recovery ( uint32_t  ui32BrickKey)

This function erases main MRAM + customer INFO space.

Parameters
ui32BrickKey- The Brick key.

This function erases main MRAM and customer INFOinstance even if the customer INFO space is programmed to not be erasable. This function completely erases the MRAM main and info instance and wipes the SRAM. Upon completion of the erasure operations, it does a POI (power on initialization) reset.

Note
This function enforces 128 bit customer key lock. The caller needs to assert the Recovery Lock using am_hal_security_set_key() providing appropriate key. Otherwise, the function will fail. Therefore, always check for a return code. If the function returns, a failure has occured.
Returns
Does not return if successful. Returns failure code otherwise. Failing return code indicates: 0x00000001 ui32BrickKey is invalid. 0x00000002 Recovery key lock not set. Other values indicate Internal error.
